# Roadmap

This roadmap outlines the current development direction for IT Dashboard. Items may change based on testing, project scope, and community feedback.

## Completed Foundation

- [x] Select Python and PyWebView as the initial application stack.
- [x] Define the frontend and backend project structure.
- [x] Create the initial desktop application shell.
- [x] Establish a frontend-to-backend API bridge.
- [x] Add starter documentation, licensing, and dependency management.

## v0.3.0 — System Information Foundation

- [x] Add a backend system-information module.
- [x] Return structured system information through the API layer.
- [x] Display operating-system, computer-name, architecture, and Python-version.
- [x] Create the first reusable dashboard information card.
- [x] create basic CSS

## Phase 2 — Core Features

- [x] Hardware information.
- [x] Performance monitoring.
- [x] Network information.
- [x] Storage information.
- [ ] Services and event-log exploration.
- [ ] Basic window navigation.
- [ ] Establish automated testing practices.

## Phase 3 — Expansion

- [ ] Additional troubleshooting tools.
- [ ] Interface customization.
- [ ] Accessibility improvements.
- [ ] Plugin-system research and design.

## Future Ideas

- Public plugin API.
- Community plugins.
