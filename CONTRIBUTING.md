# Contributing

IT Dashboard is still in early development. Contributions should support the project mission while keeping the codebase focused and maintainable.

## Before Contributing Code

1. Check whether the idea supports the project mission and design principles.
2. Open an issue before starting large features or architectural changes.
3. Keep changes focused and explain what problem they solve.
4. Consider performance, usability, maintainability, accessibility, and security.
5. Add or update documentation and tests when applicable.

## Development Setup

1. Fork or clone the repository.
2. Install Python on Windows.
3. Install the required dependency:

   ```bash
   python -m pip install -r requirements.txt
   ```

4. Run the application from the repository root:

   ```bash
   python src/backend/main.py
   ```

5. Test the affected feature before submitting a change.

## Pull Requests

- Use a clear title that summarizes the change.
- Explain the problem, the solution, and how the change was tested.
- Avoid combining unrelated changes in one pull request.
- Update relevant documentation when behavior or architecture changes.
